# Project Notes

## `ISSUES.md`

- `ISSUES.md` tracks known issues grouped by severity or disposition.
- Keep section headings stable, including empty sections such as `Major Issues`,
  `Minor Issues`, and `Ignored Issues`.
- Do not remove an issue just because code was changed. Remove an issue only
  after the user confirms that the issue is fixed.
- When removing confirmed fixed issues, renumber the remaining issues in that
  section.

## `NOTES.md`

- `NOTES.md` records project decisions, rationale, and constraints that should
  guide future changes.
- Add notes when a user explicitly asks for durable guidance or when preserving
  a decision would prevent reintroducing a known problem.
